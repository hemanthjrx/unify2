Create a `brain.md` file at the root of the project.

This file is NOT user documentation. It is persistent memory for future AI coding agents working on this repository.

The goal is that any new AI agent should be able to understand the project in under 2 minutes without scanning the entire codebase.

Analyze the complete project and continuously update this file.

The file should be structured with the following sections.

# Project Overview

- What this project is.
- Overall purpose.
- Primary users.
- Main features.
- Current development status.

List every important technology used.

Examples:

- Frontend
- Backend
- Database

- Authentication
- State management
- Styling
- Build tools
- Package manager
- File structure conventions

Explain why they exist.

# Update Policy

At the end of every coding task, automatically update this brain.md if anything important changed.

Never let this file become outdated.
